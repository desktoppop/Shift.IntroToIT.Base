Стиль коммитов

fix: Исправление ошибок
chore: Настройка среды конфигурации и т. п.
refactor: Переработка кода без изменений процессов кода
docs: Связано с документацией
feat: Добавление возможностей
delete: Удаление файлов
move: Перемещение файлов


Команды git
Clone клонирование репозитория
push отправка файлов
commit коментирование версии проекта
add добавление файла
merge слияние веток
remote подключение к удалённому репозиторию
pull Обновление файлов с репозитория

При редактирования кода необходимо создать копию и работать с ней и только при надобности слить в главную ветку и только в том случае, если не было изменений главной ветки, иначе сделать обновление ветки, после слить с главной

В тайге происходит управление проектами, создаются цели, передаются соучастникам в проекте для исполнения, обозначается текущие состояния целей (не в процессе, в процессе, готово и т. д.)
